* Mental Model: The Sieve of Truth
Imagine a grid of numbers. To find primes, we don't look for "primeness"; we look for *compositeness*.
1. We start at 2 (the first prime).
2. We "kick out" all its multiples ($4, 6, 8, \dots$) because they can't be prime.
3. We move to the next "surviving" number and repeat.
Whatever is left standing must be prime.



* 1. Basic Primality Test (Trial Division)
To check if a single number $N$ is prime, we only need to check divisors up to $\sqrt{N}$.

** The Math
If $N = a \times b$, then at least one of the factors ($a$ or $b$) must be less than or equal to $\sqrt{N}$. If we find no divisors up to $\sqrt{N}$, none exist above it.

** Implementation
#+BEGIN_SRC cpp
  bool isPrime(int n) {
      if (n <= 1) return false;
      if (n <= 3) return true;
      if (n % 2 == 0 || n % 3 == 0) return false;

      for (int i = 5; i * i <= n; i = i + 6) {
          if (n % i == 0 || n % (i + 2) == 0)
              return false;
      }
      return true;
  }
#+END_SRC

** Complexity
- *Time:* $O(\sqrt{N})$
- *Space:* $O(1)$

* 2. Sieve of Eratosthenes (Range Primality)
When you need to know all primes up to $N$ (e.g., $N = 10^6$), calling ~isPrime~ $N$ times would be $O(N\sqrt{N})$, which is too slow. The Sieve is much faster.

** Implementation
#+BEGIN_SRC cpp :results output
  #include <iostream>
  #include <vector>

  void sieve(int n) {
      // Create a boolean vector "isPrime[0..n]" and initialize
      // all entries it as true.
      std::vector<bool> is_prime(n + 1, true);
      is_prime[0] = is_prime[1] = false;

      for (int p = 2; p * p <= n; p++) {
          // If is_prime[p] is not changed, then it is a prime
          if (is_prime[p]) {
              // Update all multiples of p starting from p*p
              // Anything before p² is already handled by smaller primes.
              for (int i = p * p; i <= n; i += p)
                is_prime[i] = false;
          }
      }

      // Print first few primes for verification
      for (int p = 2; p <= 20; p++) {
          if (is_prime[p]) std::cout << p << " ";
      }
  }

  int main() {
      sieve(100);
      return 0;
  }
#+END_SRC

* Complexity Analysis

** Trial Division
- *Time:* $O(\sqrt{N})$. Every time $N$ increases by $100\times$, the time only increases by $10\times$.

** Sieve of Eratosthenes
- *Time:* $O(N \log \log N)$. This is almost linear ($O(N)$). The $\log \log N$ comes from the sum of the reciprocals of primes (divergent harmonic series of primes).
- *Space:* $O(N)$ to store the boolean array.

* Memory Anchors (The "Forget-Me-Nots")
- *The Smallest Prime:* 2 is the only even prime.
- *The 1 Trap:* *1 is NOT prime.* Always handle this in your base case.
- *Optimization:* In the sieve, the inner loop starts at ~i = p * p~. Why? Because multiples of ~p~ smaller than ~p * p~ (like $2p, 3p$) have already been marked by smaller primes.
- *Constraints:* If $N > 10^7$, a standard sieve might hit memory limits. Consider a *Segmented Sieve* or *Bitset* for $10^8$.

* 3. Odd-only Sieve (Memory Optimization)
Since all prime numbers except $2$ are $odd$, we can ignore even numbers entirely in our boolean array. This reduces the space complexity by half.

** The Mapping Logic
We map an index ~i~ in our boolean array to the odd number ~2i + 1~.
- Index 0 $\to$ 1 (Not prime)
- Index 1 $\to$ 3
- Index 2 $\to$ 5
- Index ~i~ $\to$ ~2i + 1~

** Implementation
#+BEGIN_SRC cpp :results output
  #include <iostream>
  #include <vector>

  void oddSieve(int n) {
      // Only need space for n/2 numbers
      std::vector<bool> is_prime((n / 2) + 1, true);
      is_prime[0] = false; // 1 is not prime

      for (int i = 1; (2 * i + 1) * (2 * i + 1) <= n; i++) {
          if (is_prime[i]) {
              int p = 2 * i + 1;
              // Mark odd multiples starting from p*p
              // We increment by p because (odd + odd) = even (skip),
              // but (odd + 2*odd) = odd (next multiple to check)
              for (int j = (p * p) / 2; j <= n / 2; j += p)
                  is_prime[j] = false;
          }
      }

      std::cout << "2 "; // Handle the only even prime
      for (int i = 1; i <= n / 2; i++) {
          if (is_prime[i]) std::cout << 2 * i + 1 << " ";
      }
  }

  int main() {
      oddSieve(30);
      return 0;
  }
#+END_SRC

** Complexity
- *Time:* $O(N \log \log N)$ (Similar to standard Sieve but with fewer operations).
- *Space:* $O(N/2)$ or $O(N/16)$ if using ~std::vector<bool>~ or bitsets.

* 4. Segmented Sieve (Range Optimization)
When you need to find primes in a range $[L, R]$ where $R$ is very large (e.g., $10^{12}$), but the range length $(R - L)$ is small (e.g., $10^6$), a standard sieve is impossible because you cannot allocate an array of size $10^{12}$.

** The Strategy
1. Precompute all primes up to $\sqrt{R}$ using a standard sieve.
2. Create a boolean array of size $(R - L + 1)$.
3. For each precomputed prime ($p \le \sqrt{R}$), mark its multiples within the range $[L, R]$.
4. Mark all multiples of $p$ in that range as *not prime*.

** Implementation
#+BEGIN_SRC cpp :results output
  #include <iostream>
  #include <vector>
  #include <cmath>
  #include <algorithm>

  void segmentedSieve(long long L, long long R) {
      long long limit = sqrt(R);
      std::vector<bool> mark(limit + 1, true);
      std::vector<long long> primes;

      // Step 1: Standard Sieve up to sqrt(R)
      for (long long p = 2; p <= limit; p++) {
          if (mark[p]) {
              primes.push_back(p);
              for (long long i = p * p; i <= limit; i += p)
                  mark[i] = false;
          }
      }

      // Step 2: Sieve the range [L, R]
      std::vector<bool> is_prime_range(R - L + 1, true);
      if (L == 1) is_prime_range[0] = false;

      for (long long p : primes) {
          // Find the first multiple of p in range [L, R]
          long long first_multiple = (L / p) * p;
          if (first_multiple < L) first_multiple += p;

          // Start marking from the max of (p*p) and first_multiple
          for (long long j = std::max(p * p, first_multiple); j <= R; j += p)
              is_prime_range[j - L] = false;
      }

      // Step 3: Collect results
      for (long long i = 0; i <= R - L; i++) {
          if (is_prime_range[i]) std::cout << L + i << " ";
      }
  }

  int main() {
      segmentedSieve(100, 130);
      return 0;
  }
#+END_SRC

** Complexity
- *Time:* $O((R - L + 1) \log \log R + \sqrt{R} \log \log \sqrt{R})$.
- *Space:* $O((R - L + 1) + \sqrt{R})$.

* Memory Anchors (The "Forget-Me-Nots")
- *The Index Shift:* In the Segmented Sieve, number ~X~ is stored at ~is_prime_range[X - L]~. This is the most common place for "off-by-one" errors.
- *Starting Position:* In the inner loop, ensure you don't mark the prime itself if it happens to be in the range (e.g., if $L=2, p=2$, don't mark 2 as false). Using ~std::max(p * p, first_multiple)~ handles this.
- *Memory Limits:* If $(R-L)$ is $10^6$, the boolean vector takes ~1MB~. If $R$ is $10^{12}$, $\sqrt{R}$ is $10^6$, which also takes ~1MB~. This fits comfortably in memory.
