* Mental Model: The Bidirectional Train
Unlike a Singly Linked List, each node in a DLL has two hands: one reaching forward (~next~) and one reaching backward (~prev~).
- *Pros:* Can traverse in both directions; $O(1)$ deletion if you have the node pointer (no need to find the predecessor).
- *Cons:* More memory per node; double the pointer logic (easy to mess up).



* The Structure
#+BEGIN_SRC cpp
  struct Node {
      int data;
      Node* next;
      Node* prev;
      Node(int val) : data(val), next(nullptr), prev(nullptr) {}
  };
#+END_SRC

* Core Operations Visualized
1. *Insertion:* You must update *four* pointers for a middle insertion.
2. *Deletion:* You must connect the ~prev~ of the next node to the ~next~ of the previous node.



* Interactive Lab: Insert at Tail
This demonstrates the importance of updating the ~prev~ link.

#+BEGIN_SRC cpp
  #include <iostream>

  struct Node {
      int data;
      Node *next, *prev;
      Node(int val) : data(val), next(nullptr), prev(nullptr) {}
  };

  void insertAtTail(Node*& head, int val) {
      Node* newNode = new Node(val);
      if (!head) {
          head = newNode;
          return;
      }
      Node* temp = head;
      while (temp->next) temp = temp->next;

      temp->next = newNode;
      newNode->prev = temp; // CRITICAL STEP
  }

  void printForward(Node* head) {
      while (head) {
          std::cout << head->data << " <-> ";
          if (!head->next) std::cout << "NULL";
          head = head->next;
      }
      std::cout << std::endl;
  }

  int main() {
      Node* head = nullptr;
      insertAtTail(head, 10);
      insertAtTail(head, 20);
      insertAtTail(head, 30);
      printForward(head);
      return 0;
  }
#+END_SRC

#+RESULTS:
: 10 <-> 20 <-> 30 <-> NULL

* Recursive vs Iterative Reversal
In a DLL, "reversing" actually just means swapping the ~next~ and ~prev~ pointers for every single node.

** The Recursive Approach
#+BEGIN_SRC cpp
  Node* reverseRecursive(Node* node) {
      if (!node) return nullptr;

      // Swap next and prev
      Node* temp = node->next;
      node->next = node->prev;
      node->prev = temp;

      // If prev is now null, this was the old tail (new head)
      if (!node->prev) return node;

      // Continue with the original 'next' which is now stored in 'prev'
      return reverseRecursive(node->prev);
  }
#+END_SRC

* Memory Anchors (The "Forget-Me-Nots")
- *The Boundary Trap:* When deleting the *Tail*, the ~next~ of the new tail must be ~nullptr~. When deleting the *Head*, the ~prev~ of the new head must be ~nullptr~.
- *The NULL Check:* Always check ~if (node->next != nullptr)~ before accessing ~node->next->prev~. This is the #1 cause of Segfaults in DLLs.
- *The Dummy Node:* Just like Singly LL, a dummy node at the start (and sometimes a "sentinel" at the end) eliminates almost all ~if (head == nullptr)~ edge cases.
